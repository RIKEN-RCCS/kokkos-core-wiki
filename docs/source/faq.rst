FAQ
###

.. _join-slack-workspace:

**KokkosのSlackチャンネルに参加するにはどうすればよいですか？**
 `kokkosteam.slack.com <https://kokkosteam.slack.com>`_ において、スラックチャンネルを見つけることができます。貴方のEメールで新たなアカウントを登録してください。ホワイトリストに登録できる組織の上限に達しましたが、Kokkos Slack ワークスペースの各メンバーは、さらに人々を招待できます。Slack ワークスペースに知り合いがいない場合は、Kokkos の保守担当者に連絡してください (保守担当者のメールアドレスは LICENSE ファイルに記載されています)。

**C++20 または C++23 を使ってどのように Kokkos をコンパイルできますか？**
  cmake で Kokkos を設定する際に、フラグ ``-DCMAKE_CXX_STANDARD=20`` (または ``23``) を付けてください。 いずれの下流アプリケーションに対してもフラグが設定されていることを、確認してください。

.. _setup-debug-build:

デバッグビルドを設定するにはどうすればよいですか？
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Kokkos でデバッグビルドを設定するには、完全なシンボリックデバッグ (最も遅い) と、安全性チェックを有効にしつつ最適化を維持する「高速デバッグ」のいずれかを選択できます。
詳細については :ref:`設定ガイドのデバッグに関するセクション <debugging>` を参照してください。

1. **標準 (完全) デバッグビルド**

完全なソースレベルのデバッグ (例えば、GDB、LLDB、または cuda-gdb を使用する場合) にはこれを使用してください。

* **CMake フラグ:** ``-DCMAKE_BUILD_TYPE=Debug``

* **効果**: ホストのデバッグシンボル (``-g``) を追加し、``KOKKOS_ASSERT`` を有効にします。

* **NVCC 固有の事項:** デバイス側の完全なシンボリック情報を得るには、``-DCMAKE_CXX_FLAGS="-G"`` を手動で追加し、``CMAKE_CXX_COMPILER`` として ``nvcc_wrapper`` を使用する必要があります。

.. warning:: ``-G`` はほぼすべての GPU 最適化を無効にし、カーネルの速度を著しく低下させます。

2. **高速デバッグ**

これは開発における適切な妥協案です。最適化を有効にしたまま、内部の安全性ロジックを有効にします。

* **CMake フラグ:** ``-DCMAKE_BUILD_TYPE=RelWithDebInfo`` および ``-DKokkos_ENABLE_DEBUG=ON``

* **効果:** コードを高速に保ちつつ (``-O2``)、ホストのデバッグシンボル (``-g``) を追加し、``KOKKOS_ASSERT`` を有効にします。

* **NVCC 固有の事項:** ``-DKokkos_ENABLE_DEBUG=ON`` を設定すると、NVCC で ``-lineinfo`` が自動的に追加されます。これにより、``-G`` の大幅なパフォーマンス低下を伴わずに、プロファイラ (Nsight など) 向けのソース行情報が提供されます。「高速」な GPU デバッグに追加のフラグは必要ありません。

3. **境界チェック**

デフォルトでは、上記のいずれのオプションもオーバーヘッドが大きいため境界チェックを有効にしません。View 内の「範囲外」エラーを検出するには:

* **CMake フラグ:** ``-DKokkos_ENABLE_DEBUG_BOUNDS_CHECK=ON``
