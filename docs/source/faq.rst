FAQ
###

.. _join-slack-workspace:

**KokkosのSlackチャンネルに参加するにはどうすればよいですか？**
 `kokkosteam.slack.com <https://kokkosteam.slack.com>`_ において、スラックチャンネルを見つけることができます。貴方のEメールで新たなアカウントを登録してください。ホワイトリストに登録できる組織の上限に達しましたが、Kokkos Slack ワークスペースの各メンバーは、さらに人々を招待できます。Slack ワークスペースに知り合いがいない場合は、Kokkos の保守担当者に連絡してください (保守担当者のメールアドレスは LICENSE ファイルに記載されています)。

**C++20 または C++23 を使ってどのように Kokkos をコンパイルできますか？**
  cmake で Kokkos を設定する際に、フラグ ``-DCMAKE_CXX_STANDARD=20`` (または ``23``) を付けてください。 いずれの下流アプリケーションに対してもフラグが設定されていることを、確認してください。

.. _setup-debug-build:

**デバッグビルドを設定するにはどうすればよいですか？**
  Kokkos でデバッグビルドを設定するには、完全なシンボリックデバッグ (最も遅い) と、最適化を維持しながら安全性チェックを有効にする「高速デバッグ」のいずれかを選択できます。
詳細については、:ref:`設定ガイドのデバッグセクション <debugging>` を参照してください。

1. **標準 (完全) デバッグビルド**

   これは完全なソースレベルのデバッグ (例: GDB、LLDB、または cuda-gdb の使用) に使用します。

   * **CMake フラグ:** ``-DCMAKE_BUILD_TYPE=Debug``

   * **効果**: ホストデバッグシンボル (``-g``) を追加し、``KOKKOS_ASSERT`` を有効にします。

   * **NVCC 固有の事項:** 完全なデバイス側のシンボリック情報を得るには、``-DCMAKE_CXX_FLAGS="-G"`` を手動で追加し、``nvcc_wrapper`` を ``CMAKE_CXX_COMPILER`` として使用する必要があります。

   .. warning:: ``-G`` は、ほぼすべての GPU 最適化を無効にし、カーネルを大幅に遅くします。

2. **高速デバッグ**

   これは開発における良い妥協点です。最適化を有効に保ちつつ、内部の安全性ロジックをオンにします。

   * **CMake フラグ:** ``-DCMAKE_BUILD_TYPE=RelWithDebInfo`` および ``-DKokkos_ENABLE_DEBUG=ON``

   * **効果:** コードを高速 (``-O2``) に保ちながら、ホストデバッグシンボル (``-g``) を追加し、``KOKKOS_ASSERT`` を有効にします。

   * **NVCC 固有の事項:** ``-DKokkos_ENABLE_DEBUG=ON`` を設定すると、NVCC で ``-lineinfo`` が自動的に追加されます。これにより、``-G`` の大幅なパフォーマンス低下を伴わずに、プロファイラ (Nsight など) 向けのソース行情報が提供されます。「高速」な GPU デバッグには追加のフラグは必要ありません。

3. **境界チェック**

   デフォルトでは、上記のどちらのオプションも、高いオーバーヘッドのために境界チェックを有効にしません。View で「範囲外」エラーを検出するには:

   * **CMake フラグ:** ``-DKokkos_ENABLE_DEBUG_BOUNDS_CHECK=ON``
